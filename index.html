MADATA

A reporting platform I designed and built to replace hand-copied, vague monthly numbers with structured, source-traceable, print-ready client reports — plus AI-assisted recommendations.

The problem

Vague by hand

Reporting meant clicking into each client's social, scheduling, and display dashboards and hand-copying three or four flattering numbers — reach, followers, posts, impressions — into a document. Every client had different data, in different places, with no shared structure.

The build

One pipeline, many shapes

I built MADATA in Python / FastAPI with SQLite and Supabase, wired to the Claude API. It ingests raw export files from every source — native platform exports, scheduling tools, display, search, profile, geofence, and GA4 — parses them, and generates a monthly report per client. It's designed around the reality that no two clients share the same data shape.

The impact

Depth, then a human note

Reports went from 3–4 numbers to dozens of structured metrics and hundreds of data points each, with AI-generated recommendations. I kept a hand-written client-notes section on every report — because in a small market, the people reading them want plain, friendly guidance, not a dashboard.

SEO / AEO process

The same repeatable loop on every site: measure first, change deliberately, then prove it moved. Nothing is called a win until the data backs it.

01

Crawl & baseline

Full Screaming Frog crawl plus a GA4 and Search Console baseline, so every later change can be measured against where the site actually started.

Screaming FrogGA4Search Console
02

Research & audit

SpyFu for keyword and competitor gaps, read against the site's current pages to see where it is already winning, where it is invisible, and which terms are actually worth chasing.

SpyFuKeyword researchCompetitor research
03

Answer-engine structuring

Match the fix to the page. Where a site has no FAQ or supporting article, add one so answer engines have something to quote; where the pages already exist, tailor the schema and content changes to each one.

FAQ / article pagesSchemaPer-page AEO
04

Report & implement

Turn the findings into a deep, evidence-backed report of exactly what to change and why, then implement it directly in the CMS, front-end and back-end, and deploy Google tag and tracking code so results are captured from day one.

Recommendations reportWordPress / Wix / SquarespaceGoogle Tag
05

Measure & report

Re-baseline against GA4 and Search Console, show before and after, and flag what is verified versus still maturing. Search moves slowly, so the report says so honestly.

GA4Search ConsoleBefore / after

Paid media

My paid media is driven by the numbers, not guesswork. Before each cycle I review the previous month's performance for that specific business: which audiences, placements, and creative actually drove results and which fell flat. Then I rebuild the targeting and messaging around what worked, so every client's ads are tuned to their own data, business by business, rather than a shared template.

The build itself is a structured campaign to ad set to ad, with the objective and budget set first, then audience and creative shaped to match. The example below is anonymized demo data.
